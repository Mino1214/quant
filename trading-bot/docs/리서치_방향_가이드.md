# 전략 리서치 방향 가이드 (처음부터 차근차근)

## 1. 우리가 풀려는 문제

- **현재:** ema_distance + RSI + volume 같은 **기준(임계값)** 만으로 필터링하면, **평균적으로 돈을 못 벌는 구간**이 나온다 → **기대수익(expectancy)이 음수**.
- **목표:** “어떤 조건에서만 거래할 때” **기대수익이 양수**가 되는지 찾고, 전략을 그에 맞게 **구조부터** 바꾸는 것.

즉, **임계값만 미세 조정**하는 게 아니라, **“언제/어떤 시장/얼마나 오래 잡을지/어떤 진입만 허용할지”** 같은 **구조**를 먼저 정하는 게 방향입니다.

---

## 2. 리서치가 하는 일 (한 문장씩)

리서치는 **“지금 전략이 왜 음수인지”**를 쪼개서 보는 작업입니다.

| 질문 | 리서치 이름 | 하는 일 | 나오는 결과 |
|------|-------------|---------|-------------|
| 역추세로 들어가서 망하는 건 아닐까? | **트렌드 정렬** | “추세 방향하고 같은 방향만 진입”했을 때 지표가 나아지는지 비교 | 트렌드 필터 넣었을 때 avg_R / PF 개선 여부 |
| 특정 시장 상태에서만 이득이 나오는 건 아닐까? | **레짐 분리** | 상승 추세 / 하락 추세 / 횡보 구간을 나눠서 각각 지표 계산 | TRENDING_UP/DOWN/RANGING 중 어디서 양수인지 |
| 진입 후 **몇 봉**까지 잡을 때가 제일 나을까? | **엣지 감쇠** | 5봉/10봉/20봉/30봉 뒤 수익률을 각각 계산 | “5봉에서만 양수”, “오래 잡으면 망한다” 같은 인사이트 |
| 진입 **품질**을 올리면 나아질까? | **진입 품질 필터** | 풀백 깊이, 브레이크아웃, 봉 몸통/꼬리 등으로 “질 나쁜 진입” 제거 후 스캔 | 필터 적용 시 avg_R 개선 여부 |

정리하면:

- **트렌드 정렬** → “방향”을 맞추는지 보는 것  
- **레짐 분리** → “어떤 시장”에서만 쓸지 보는 것  
- **엣지 감쇠** → “몇 봉까지” 잡을지 보는 것  
- **진입 품질** → “어떤 진입만” 허용할지 보는 것  

이 네 가지가 **“무슨 리서치를 하는지”**에 대한 답입니다.

---

## 3. 전체 흐름 (어디서 무엇을 할지)

```
[1단계] 트렌드 정렬
   → "역추세 진입이 원인인가?" 확인
   → 트렌드 필터 넣었을 때 지표가 좋아지면: 전략에 "추세 방향만 진입" 추가

[2단계] 레짐 분리
   → "상승/하락/횡보 중 어디서 돈을 버는가?" 확인
   → 예: TRENDING_UP에서만 양수면: "상승 추세일 때만 롱" 같은 제한 추가

[3단계] 엣지 감쇠
   → "몇 봉 뒤에 수익이 가장 좋은가?" 확인
   → 예: 5봉에서만 양수면: TP/청산/최대 보유 봉을 5봉 근처로 설계

[4단계] 진입 품질
   → "질 나쁜 진입(풀백 없음, 브레이크아웃 없음 등)을 빼면 나아지는가?" 확인
   → 나아지면: 후보 생성 단계에 풀백/브레이크아웃/몸통 조건 추가

[마지막] 임계값 튜닝
   → 위 구조가 정해진 "후보 집단"에 대해서만
   → ema_distance, volume_ratio, RSI 임계값을 스캔 (보조 수단)
```

**방향성:**  
먼저 **구조(트렌드/레짐/보유기간/진입품질)** 를 정하고, 그 다음에 **임계값**을 만지는 순서입니다.

---

## 4. 지금 데이터로 알 수 있는 것 / 없는 것

- **알 수 있는 것**
  - **레짐 분리:** TRENDING_UP 구간에서 5봉·30봉 avg_R이 약간 양수. TRENDING_DOWN은 전 구간 음수.
  - **엣지 감쇠:** 5봉에서만 약한 양수, 10봉 이후로는 음수 → “오래 잡지 말고 5봉 근처에서 정리”가 타당해 보임.
- **아직 못 보는 것**
  - **트렌드 정렬:** DB에 ema20/ema50/기울기 정보가 없어서, “트렌드 필터 적용 시” 비교가 0건으로 나옴.
  - **진입 품질(브레이크아웃/풀백):** DB에 해당 필드가 없어서, 이 필터만 켜면 0건이 됨.  
  → **이걸 추가하는 방법:** [트렌드_진입품질_필드_추가하기.md](트렌드_진입품질_필드_추가하기.md) 참고 (데이터셋 다시 빌드하면 됨).

그래서 **“방향은 정해져 있는데, 일부는 데이터를 다시 만들어야 숫자로 확인 가능”**한 상태라고 보면 됩니다.

## 5. 수정을 어떻게 해나가면 되는지 (실행 순서)

### 5-1. 당장 코드 수정 없이 할 수 있는 것

- **레짐:**  
  이미 나온 레짐별 CSV/엣지감쇠 결과를 보고,  
  “TRENDING_UP + 5봉 근처 청산” 같은 **규칙**을 메모해 두기.
- **보유 기간:**  
  엣지 감쇠 결과를 보고 TP/트레일/최대 보유 봉을 **5봉 근처**로 가져가는 걸 검토.

### 5-2. “트렌드 정렬” 효과를 보려면

1. **데이터 다시 만들기**  
   - `build_signal_dataset`으로 후보를 다시 생성하면,  
     `feature_values_ext`에 **ema20, ema50, ema50_slope** 등이 들어갑니다.
2. 그다음 **스캔 다시 실행**  
   - `run_stability_scan --from-db --compare-trend`  
   - 여기서 no_trend vs trend_filtered 결과가 **0건이 아니게** 나와야, 트렌드 필터가 도움이 되는지 판단 가능.

즉, **수정 방향:**  
“트렌드 정렬 필터를 켰을 때 지표가 좋아지는가?”를 보려면 → **데이터 재구축 → 스캔 재실행** 순서로 가면 됩니다.

### 5-3. “진입 품질(브레이크아웃/풀백)” 효과를 보려면

1. **데이터 다시 만들기**  
   - 지금 코드에는 이미 **pullback_depth_pct, breakout_confirmation** 등이 feature로 들어가므로,  
     `build_signal_dataset`을 다시 돌리면 새 후보에는 이 값들이 채워집니다.
2. 그다음 **스캔에 진입 품질 옵션 적용**  
   - 예: `--require-breakout`, `--min-pullback 0.2 --max-pullback 0.8`  
   - 이때 통과하는 건수가 0이 아니면, “이 필터를 넣었을 때 avg_R이 나아지는가?”를 비교할 수 있음.

**수정 방향:**  
“브레이크아웃/풀백 필터가 도움이 되는가?”를 보려면 → **데이터 재구축(같은 스크립트로 충분) → 스캔 시 위 옵션 추가**로 가면 됩니다.

### 5-4. 전략 쪽에 반영할 수정 (구조 개선)

리서치 결과가 나온 뒤, 아래처럼 **설정/로직**을 바꾸면 됩니다.

- **트렌드 정렬이 도움이 되면**  
  - config에서 `use_trend_filter: true` 또는  
  - 승인/진입 직전에 “ema20 > ema50 & ema50_slope > 0 (롱)” 같은 체크 추가.
- **레짐이 중요하면**  
  - “TRENDING_UP일 때만 롱 허용”, “TRENDING_DOWN에서는 숏만 또는 거래 안 함” 같은 **레짐별 규칙** 추가.
- **5봉 근처 청산이 유리하면**  
  - TP 목표, 트레일링, **max_bars_in_trade** 등을 5봉 근처로 줄이기.
- **진입 품질이 도움이 되면**  
  - 후보 생성/승인 단계에 **pullback_depth_pct, breakout_confirmation** 조건 추가 (이미 코드에 필터/feature 있음).

## 6. 한 줄 요약

- **리서치가 하는 일:**  
  “지금 전략이 왜 음수인지”를 **트렌드 방향 / 시장 레짐 / 보유 기간 / 진입 품질** 네 가지로 쪼개서 측정하는 것.
- **수정을 어떻게 해나가냐:**  
  1) **데이터 재구축**으로 트렌드·진입품질 숫자를 채우고,  
  2) **스캔으로 “어떤 구조가 나은지”** 확인한 뒤,  
  3) **설정/규칙**을 그 구조에 맞게 바꾸고,  
  4) **임계값**은 그 다음에 보조적으로 튜닝.

이 순서만 따라가면 “방향을 모르겠다”는 부분은 정리되고, “다음에 뭘 수정해야 하지?”도 위 5-2, 5-3, 5-4만 보면 됩니다.
