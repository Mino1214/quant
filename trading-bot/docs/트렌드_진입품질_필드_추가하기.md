# 트렌드 정렬·진입 품질 필드를 쓰려면 (DB에 추가하는 방법)

가이드 68~70줄에서 말하는 **"아직 못 보는 것"** — 트렌드 정렬(ema20/ema50/기울기), 진입 품질(브레이크아웃/풀백) — 을 리서치에서 쓰려면 **DB의 `candidate_signals.feature_values_ext`에 해당 값이 들어가 있어야** 한다.

---

## 0. DB 컬럼 추가가 필요한가? → **아니요**

- **테이블에 컬럼을 더 만들 필요 없음.**  
  `ema20_gt_ema50`, `pullback_depth_pct` 같은 건 **DB 컬럼이 아니라**,  
  **`feature_values_ext` 한 컬럼(TEXT) 안에 들어가는 JSON의 키**다.
- `init_db()` 할 때 없으면 `_migrate_candidate_signals()`가 `feature_values_ext` 컬럼을 추가한다.  
  이미 있으면 그대로 쓴다.
- **배치를 돌리면** `build_signal_dataset` → `extract_feature_values()` → `feature_values` 전체를 JSON으로 저장하므로,  
  **이번에 새로 INSERT되는 행**에는 현재 코드 기준으로 트렌드·진입 품질 필드가 JSON 안에 다 들어간다.

**“배치 돌렸는데 (필드가) 안 생긴다”**면 보통 아래 경우다.

- **그 구간이 예전에 이미 다 쌓여 있어서** `--skip-existing` 때문에 **모든 봉이 스킵**되고, **새로 INSERT되는 행이 0건**인 경우.  
  → 그 구간을 **다시** 채우려면, **해당 구간 기존 후보(·아웃컴)를 삭제한 뒤** 배치를 다시 돌려야 한다.  
  [아래 3-2 참고](#3-2-이미-쌓인-구간을-새-필드로-다시-채우려면)

---

## 1. 왜 지금 0건인가

- **코드는 이미 구현되어 있음.**  
  `strategy/feature_extractor.py`에서 ema20, ema50, ema50_slope, pullback_depth_pct, breakout_confirmation 등을 계산하고,  
  `scripts/build_signal_dataset.py`가 후보를 DB에 넣을 때 `feature_values` 전체를 JSON으로 `feature_values_ext`에 저장한다.
- **그런데** 지금 DB에 있는 후보 행들은 **예전에** 쌓인 것이어서, 그때는 이 필드들이 없었거나, 예전 버전으로 쌓여서 **feature_values_ext 안에 이 키들이 없다.**
- 그래서 스캔에서 "트렌드 필터"나 "진입 품질 필터"를 쓰면, 조건을 만족하는 행이 0건이 된다.

---

## 2. 추가하려면 어떻게 해야 하나 (한 줄 요약)

**코드 수정은 필요 없다.**  
**원하는 기간에 대해 후보 시그널을 다시 만드는 스크립트(`build_signal_dataset`)를 한 번 더 실행**하면, 그때 **새로 INSERT되는 행**에는 트렌드·진입 품질 필드가 전부 들어간다.

- 이미 DB에 있는 **옛 행**은 과거 캔들 데이터가 없어서 나중에 값을 채울 수 없다.  
- 그래서 **“추가”는 = 새로 빌드한 구간의 데이터가 쌓이게 하는 것**이라고 보면 된다.

---

## 3. 실제로 할 작업 (순서)

### 3-1. 후보 시그널 데이터셋 다시 만들기

프로젝트 루트(`trading-bot`)에서:

```bash
# 예: 2024년 구간만 새로 빌드 (이미 있는 타임스탬프는 --skip-existing 로 스킵)
python -m scripts.build_signal_dataset --symbol BTCUSDT --start 2024-01-01 --end 2024-12-31 --skip-existing
```

또는 배치 스크립트를 쓰는 경우:

```bash
python3 scripts/build_signal_dataset_batch.py --from 2024-01-01 --to 2024-12-31 --workers 4 --chunk-months 6
```

- **주의:** 1m 캔들이 DB(예: `btc1m`)에 그 기간만큼 있어야 한다.  
- 이 단계가 끝나면, **새로 쌓인 후보 행**의 `feature_values_ext` 안에  
  `ema20`, `ema50`, `ema20_gt_ema50`, `ema50_slope`, `pullback_depth_pct`, `breakout_confirmation`, `lower_wick_ratio`, `upper_wick_ratio` 등이 들어가 있다.

### 3-2. 빌드가 끝난 뒤 스캔 다시 실행

- **트렌드 정렬 효과 보기**  
  ```bash
  python -m analysis.run_stability_scan --from-db --compare-trend --limit 20000
  ```  
  → 이제 트렌드 필터 적용 시 0건이 아니면, no_trend vs trend_filtered 비교 가능.

- **진입 품질(브레이크아웃/풀백) 효과 보기**  
  ```bash
  python -m analysis.run_stability_scan --from-db --require-breakout --min-pullback 0.2 --max-pullback 0.8
  ```  
  → 통과하는 건수가 0이 아니면, 해당 필터가 avg_R에 미치는 영향을 비교할 수 있다.

---

## 4. 정리

| 하고 싶은 것           | 필요한 작업 |
|------------------------|------------|
| 트렌드 정렬 필터 비교  | 1) 해당 기간으로 `build_signal_dataset` 실행 → 2) `run_stability_scan --from-db --compare-trend` |
| 진입 품질 필터 비교   | 1) 해당 기간으로 `build_signal_dataset` 실행 → 2) `run_stability_scan --from-db --require-breakout` 등 옵션으로 스캔 |

**“이걸 추가하려면 어떻게 해야 하지?”** → **해당 기간에 대해 `build_signal_dataset`(또는 배치)을 한 번 더 돌려서 새 후보를 쌓으면 된다.**
